Rendering and support comments

All browsers tested supported SVG and its <text> element to some extent. Firefox rendered the same file much larger than Epiphany. Konqeror made a very small but spaced out rendering of the different text elements.

  • To my eyes, Firefox and Epiphany rendered the Serif font family the same as each other, but differently to Konqueror. For Serif, it appeared to me that all browsers rendered it in SVG the same as they did unstyled HTML.
  • In Garamond, only Epiphany seemed to do something different to what it did with Serif.
  • Arial I think was correctly supported by Firefox and Epiphany, while Konqueror continued to render it the same as default web fonts (which may well have been Arial in a stroke of luck).
  • Monospace was clearly a monospace font face of sorts on Firefox and Epiphany, but not rendered differently in Konqueror.
  • The Cursive font family was quite well spaced in Firefox and Epiphany, but rendered like Arial in Firefox and Serif in Epiphany. Konqueror did its usual default rendering. I don't think it supports text fonts in SVG.
  • Courier was quite like I know it in both Firefox and Epiphany and not supported in Konqueror.
  • A chosen additional font family, Verdana, rendered in Firefox and Epiphany like each of them rendered Garamond, which was very different to each other. Konqueror failed again.
  • Palatino, my next selected font, looked very similar between Firefox and Epiphany and unlike either browser had rendered any other font family in this list. Konqueror did not render it differently.
